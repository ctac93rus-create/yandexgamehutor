## Milestone 1: Merge Loop
- Игрок работает на доске 6x7: перемещает предметы, совмещает одинаковые tier-предметы, раскрывает следующую стадию цепочки.
- Генератор (например костёр) тратит заряды на выдачу базового ресурса, заряды восстанавливаются по таймеру.
- При заполненной доске применяется overflow-политика: выдача безопасной компенсации в пыли, чтобы не возникал софтлок.
- UI: верхняя панель (золото/пыль), нижняя навигация (меню/хутор/рейд), тосты для обратной связи.

## Milestone 2: Raid MVP
- Длительность сессии: 30–90 секунд (в текущем контенте 55 секунд) с 3 линиями атаки.
- Волны описаны в `waves.json`: таймлайн появления врагов по lane + enemyId.
- Враги двигаются к базе; каждый дошедший враг снимает 1 HP базы. При 0 HP — поражение без зависаний.
- По каждой линии стоит защитный тотем (auto-attack, DPS/range) и стреляет в ближайшую цель.
- Активный спелл: Freeze с кнопкой и кулдауном. Эффект по области, применяет заморозку.
- По итогам рейда показывается reward screen: «Забрать» и «Удвоить (rewarded)».
- Награда переносится в merge: валюта + предметы; при нехватке мест срабатывает OverflowPolicy.
- Автосейв выполняется при завершении рейда и после применения наград в merge.

## Protocol A–E (Raid)
- A. `Waves` парсит и валидирует контент, сортирует таймлайн.
- B. `RaidController` ведёт состояние `running/victory/defeat`.
- C. `Enemy/Defender/Spells` отвечают за бой и производительность update-loop.
- D. `RewardCalculator` считает базовую/удвоенную награду.
- E. `RewardModal` завершает цикл через claim в MergeScene.
